# QMT / PTrade 实盘交易集成研究

本文档介绍如何将A股量化交易模拟系统与实盘交易接口（QMT / PTrade）集成，实现从模拟到实盘的过渡。

---

## 1. QMT（迅投极速交易系统）

### 1.1 概述
QMT（Quick Model Trade）是迅投科技开发的极速交易系统，广泛应用于国内量化交易。连接券商QMT服务器，提供Python API，可实现下单、撤单、查持仓、查资金等操作。

### 1.2 工作原理
- 用户需在券商开通QMT权限（如国金证券、银河证券、华泰证券等多数主流券商均支持）
- 下载迅投miniQMT软件（仅支持Windows系统）
- miniQMT作为客户端，连接券商QMT服务器
- 通过Python API（`Xtptrader`/`Xtdata`等库）发送交易指令
- API运行在用户本地Windows机器上

### 1.3 核心API函数

```python
fromXtPython import *

# 初始化
api = XtCenterApi()
api.init()

# 登录
api.login(account="您的账户", password="密码", server="服务器地址")

# 下单
# symbol: 证券代码，如 "688318.SH"
# price: 价格（0为市价）
# volume: 数量（必须是100的整数倍）
# order_type: 23=买入, 24=卖出
api.order(symbol="688318.SH", price=100.0, volume=100, order_type=23)

# 撤单
api.cancel_order(order_id="订单号")

# 查持仓
positions = api.get_positions()
# 返回: [{'证券代码': '688318', '持仓数量': 500, '成本价': 98.5, ...}]

# 查资金
account = api.get_account()
# 返回: {'可用资金': 500000.00, '总资产': 1500000.00, ...}

# 查当日成交
trades = api.get_today_trades()
# 返回: [{'成交时间': '09:30:15', '证券代码': '688318', '成交数量': 100, '成交价格': 100.5, ...}]
```

### 1.4 入门步骤
1. **选择支持QMT的券商**（如国金证券、银河证券等）
2. **申请QMT权限**（通常要求资产10万以上，部分券商门槛更低）
3. **下载迅投miniQMT客户端**（官网或券商提供的下载链接）
4. **安装并登录**（使用券商提供的账号）
5. **编写Python策略**（使用`xtquant`等第三方库）
6. **回测验证后实盘测试**（先跑模拟盘，再切实盘）

### 1.5 局限性
- **仅支持Windows系统**（miniQMT客户端无Linux/Mac版）
- **需要券商支持**（并非所有券商都提供QMT）
- **本地运行**（策略必须运行在本地Windows机器上，无法云端部署）
- **网络稳定性依赖本地**（断网则策略中断）
- **权限门槛**（通常需要账户资产10万以上）

## 2. PTrade（恒生Ptrade）

### 2.1 概述
PTrade是恒生电子开发的云端量化交易平台，支持浏览器开发策略，Python API丰富，覆盖多数主流券商。

### 2.2 工作原理
- 云端托管策略，用户无需本地运行（浏览器登录即可）
- 券商开通PTrade权限后，获得云端策略编辑环境
- 内置Python API，支持下单、查持仓、查资金、算法交易等
- 支持回测、模拟盘、实盘一站式开发
- 部分券商支持将策略部署在云端服务器，7x24小时运行

### 2.3 核心API函数

```python
# PTrade API（通过内置对象访问）
import PTrade

# 下单
# code: 证券代码，如 "688318.SH"
# price: 价格（0为市价）
# amount: 数量（必须是100的整数倍）
# entrust_type: 1=买入, 2=卖出
PTrade.order(entrust_no="", code="688318.SH", price=100.0, amount=100, entrust_type=1)

# 撤单
PTrade.cancel_order(entrust_no="订单号")

# 查持仓
positions = PTrade.get_positions()

# 查资金
account = PTrade.get_account()

# 查当日委托
orders = PTrade.get_today_orders()

# 查当日成交
trades = PTrade.get_today_trades()
```

### 2.4 入门步骤
1. **选择支持PTrade的券商**（如银河证券、华宝证券、国金证券等）
2. **申请PTrade权限**（通过券商客户经理申请，通常需要资产10万+）
3. **登录云端平台**（浏览器访问券商提供的PTrade地址）
4. **编写策略**（使用内置Python编辑器，或本地写好上传）
5. **模拟盘验证**（先用模拟账号测试）
6. **实盘部署**（审核后切换实盘）

### 2.5 局限性
- **云端策略需要审核**（部分券商对策略有合规审查）
- **编程环境受限**（依赖券商提供的API，功能有所限制）
- **无法连接自己编写的数据源**（行情来源固定）
- **网络延迟**（云端部署vs本地，略有延迟差异）

## 3. QMT vs PTrade 对比

| 对比项 | QMT | PTrade |
|--------|-----|--------|
| **操作系统** | 仅Windows | 云端/浏览器（跨平台） |
| **部署方式** | 本地运行 | 云端托管（可选本地客户端） |
| **编程语言** | Python / VBA | Python |
| **策略隐私** | 本地存储，隐私性强 | 云端存储，需信任券商 |
| **券商支持** | 多数主流券商 | 多数主流券商 |
| **资产门槛** | 10万+ | 10万+ |
| **费用** | 免费/低佣金 | 免费/低佣金 |
| **网络要求** | 本地网络稳定 | 云端，网络要求低 |
| **自定义行情** | 支持接入各类数据源 | 行情源固定 |
| **运行稳定性** | 依赖本地电脑 | 云端服务器，较稳定 |
| **算法交易** | 支持（自行对接） | 内置算法交易模块 |

**选择建议**：
- 如果需要**完全本地控制**、**策略高度自定义**、接入自有数据源 → 选择 **QMT**
- 如果希望**省心省力**、**随时随地访问**、不想维护本地电脑 → 选择 **PTrade**

## 4. 与本模拟系统的集成方案

本系统当前为**纸盘模拟**（paper trading），交易信号需要用户手动执行。若要实现自动化实盘交易，有以下几种集成路径：

### 方案A：QMT本地Python对接（本系统推荐）

**架构**：
```
本系统（Linux服务器）
    ↓ 生成交易信号
    ↓ REST API / 文件共享
QMT客户端（Windows虚拟机/物理机）
    ↓ 调用XtQuant API
券商QMT服务器
    ↓ 实盘成交
```

**实施步骤**：
1. 在Windows机器上安装miniQMT客户端并登录
2. 在Windows上运行一个**信号监听脚本**（轮询本系统API或文件）
3. 脚本接收到信号后，调用QMT API执行下单
4. 下单结果回传至本系统，更新记录

**优点**：完全本地控制，策略私密性好
**缺点**：需要维护Windows机器，网络需打通

### 方案B：PTrade云端对接

**架构**：
```
本系统（Linux服务器）
    ↓ REST API
PTrade云端策略
    ↓ 恒生API
券商服务器
    ↓ 实盘成交
```

**实施步骤**：
1. 申请PTrade云端策略权限
2. 在PTrade中编写策略，通过API获取本系统的信号
3. PTrade执行实盘交易

**优点**：无需维护本地机器，云端稳定运行
**缺点**：策略逻辑需提交给券商，隐私性较低

### 方案C：维持纸盘 + 手动执行（当前方案，最稳妥）

**架构**：
```
本系统生成信号 → 飞书推送通知 → 用户手动操作券商App执行
```

**优点**：
- 无需额外软件/硬件投入
- 完全无风险（不会因为程序错误导致意外亏损）
- 符合监管要求（用户自主决策和操作）

**缺点**：
- 无法自动化，依赖人工执行
- 延迟和情绪因素影响执行效果

**适合人群**：策略验证阶段、资金量较小、风险偏好较低的用户

## 5. 推荐的实盘过渡路径

1. **第一阶段（当前）**：使用纸盘模拟，跟踪策略表现，验证策略有效性
2. **第二阶段（小资金实盘）**：选择支持QMT或PTrade的券商，用小资金（1-2万）跑实盘，验证策略在实际成交中的表现
3. **第三阶段（自动化）**：对接QMT/PTrade API，实现信号→下单全自动
4. **第四阶段（组合管理）**：扩展到多策略、多标的，实现组合量化交易

---

## 6. 风险提示

- **量化交易存在风险**，历史回测表现不代表未来收益
- **实盘交易可能面临滑点、流动性、涨跌停等限制**
- **程序化交易需遵守监管规定**，不得进行异常交易行为
- **建议先在模拟盘充分验证后再进入实盘**
